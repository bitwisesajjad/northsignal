# Data Dictionary

## OpenCelliD — Finland Tower Data

Source: OpenCelliD (Unwired Labs)
File: `data/raw/opencellid_finland.csv`
Rows: 33,063
Filter: MCC 244 (Finland only)

| Column        | Description                                                                       |
| ------------- | --------------------------------------------------------------------------------- |
| radio         | Network technology: GSM (2G), UMTS (3G), LTE (4G), NR (5G)                        |
| mcc           | Mobile Country Code. 244 for Finland                                              |
| net           | Mobile Network Code (MNC). Identifies the operator: 1 = Elisa, 3 = DNA, 5 = Telia |
| area          | Location Area Code (LAC). Groups cells into geographic areas within a network     |
| cell          | Cell ID. Unique identifier for a single antenna within an area                    |
| unit          | Secondary cell identifier, used in some network types. Usually 0                  |
| lon           | Longitude of the tower in decimal degrees                                         |
| lat           | Latitude of the tower in decimal degrees                                          |
| range         | Estimated signal range of the tower in meters                                     |
| samples       | Number of measurements used to calculate this tower's position                    |
| changeable    | Whether the tower position can be updated by new measurements. 1 = yes            |
| created       | Unix timestamp when this tower was first added to the database                    |
| updated       | Unix timestamp when this tower record was last updated                            |
| averageSignal | Average signal strength across all measurements for this tower                    |

## FMI (Finnish Meteorological Institute) — Oulu Weather Data

Source: FMI Open Data WFS API
Station: Oulu (fmisid 101794), coordinates 65.00637, 25.39325
File: `data/raw/fmi_oulu_2022_2025.csv`
Coverage: 2022-01-01 to 2025-12-31, one observation every 10 minutes

| Column        | Description                                                                                                       |
| ------------- | ----------------------------------------------------------------------------------------------------------------- |
| time          | UTC timestamp of the observation in ISO 8601 format                                                               |
| temperature_c | Air temperature in degrees Celsius (FMI parameter: t2m)                                                           |
| snow_depth_cm | Snow depth in centimetres measured by automatic sensor (FMI parameter: snow_aws). Null when sensor had no reading |
| wind_speed_ms | Average wind speed in metres per second over the preceding 10 minutes (FMI parameter: ws_10min)                   |

## DIM_TOWERS — Generated Tower Dimension

Source: Generated by `src/generator/generate_dim_towers.py`
File: `data/generated/dim_towers.csv`
Rows: 8,761
Input: `data/raw/opencellid_finland.csv`, Statistics Finland municipality boundaries (WFS API)

| Column         | Type    | Description                                                                                            |
| -------------- | ------- | ------------------------------------------------------------------------------------------------------ |
| tower_id       | VARCHAR | Surrogate key in format TWR-00001. Primary key for this dimension                                      |
| latitude       | FLOAT   | Tower latitude in decimal degrees (WGS84). Sourced directly from OpenCelliD                            |
| longitude      | FLOAT   | Tower longitude in decimal degrees (WGS84). Sourced directly from OpenCelliD                           |
| municipality   | VARCHAR | Finnish municipality name derived via spatial join against Statistics Finland boundary polygons        |
| region         | VARCHAR | Finnish region name derived from municipality. Finland has 19 regions                                  |
| operator       | VARCHAR | Mobile operator name. One of: Elisa, DNA, Telia. Derived from OpenCelliD MNC field                     |
| technology     | VARCHAR | Network generation. One of: 4G, 5G. Derived from OpenCelliD radio field (LTE=4G, NR=5G)                |
| frequency_band | VARCHAR | Spectrum band assigned to this tower. Based on real Traficom Finnish spectrum allocations per operator |
| mast_height_m  | FLOAT   | Mast height in metres. Sampled from realistic ranges: urban 15-30m, suburban 25-40m, rural 35-55m      |

Source: Generated by `src/generator/generate_dim_customers.py`
File: `data/generated/dim_customers.csv`
Rows: 50,000
Input: IBM Watson Telco Churn dataset distributions

| Column            | Type    | Description                                                                                                          |
| ----------------- | ------- | -------------------------------------------------------------------------------------------------------------------- |
| customer_id       | VARCHAR | Surrogate key in format CUST-00001. Primary key for this dimension                                                   |
| contract_type     | VARCHAR | Contract length. One of: month_to_month, one_year, two_year. ~55% month-to-month                                     |
| device_type       | VARCHAR | Device category. One of: smartphone, tablet, mobile_router, iot_device. ~70% smartphone                              |
| subscription_tier | VARCHAR | Service tier. One of: basic, standard, premium. ~50% standard                                                        |
| churn_risk_score  | FLOAT   | Churn probability score from 0.0 to 1.0. Derived from IBM churn distributions. Month-to-month customers score higher |

File: `data/generated/dim_time.csv`

Coverage: 2022-01-01 00:00:00 to 2025-12-31 23:50:00, every 10 minutes

| Column     | Type      | Description                                                           |
| ---------- | --------- | --------------------------------------------------------------------- |
| time_id    | VARCHAR   | Surrogate key in format TIME-0000001. Primary key for this dimension  |

| hour       | INTEGER   | Hour of day. 0 to 23                                                  |
| day        | INTEGER   | Day of month. 1 to 31                                                 |
| week       | INTEGER   | ISO week number. 1 to 53                                              |
| month      | INTEGER   | Month number. 1 to 12                                                 |
| year       | INTEGER   | Calendar year. 2022 to 2025                                           |
| is_weekend | BOOLEAN   | True if the timestamp falls on a Saturday or Sunday                   |

## DIM_WEATHER — Generated Weather Dimension

Source: Generated by `src/generator/generate_dim_weather.py`
File: `data/generated/dim_weather.csv`
Rows: 2,101,907
Input: `data/raw/fmi_oulu_2022_2025.csv`

| Column        | Type      | Description                                                                                                                         |
| ------------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| weather_id    | VARCHAR   | Surrogate key in format WTH-0000001. Primary key for this dimension                                                                 |
| timestamp     | TIMESTAMP | UTC timestamp of the observation. Used to join to DIM_TIME and FACT_NETWORK_EVENTS during generation. Not in final warehouse schema |
| temperature_c | FLOAT     | Air temperature in degrees Celsius. Range -30.3 to 30.8 in the Oulu dataset                                                         |
| snow_depth_cm | FLOAT     | Snow depth in centimetres. Null when sensor had no reading, typically in summer                                                     |
| wind_speed_ms | FLOAT     | Average wind speed in metres per second over the preceding 10 minutes                                                               |
| condition     | VARCHAR   | Derived weather label. One of: clear, windy, snow, blizzard. Derived from temperature, snow depth, and wind speed                   |

Condition derivation thresholds:

| Condition | Rule                                               |
| --------- | -------------------------------------------------- |
| blizzard  | wind >= 10.0 m/s AND snow >= 10.0 cm               |
| snow      | snow >= 2.0 cm (and wind below blizzard threshold) |
| windy     | wind >= 7.0 m/s (and snow below snow threshold)    |
| clear     | all other conditions                               |

## DIM_INCIDENTS — Generated Incidents Dimension

Source: Generated by `src/generator/generate_dim_incidents.py`
File: `data/generated/dim_incidents.csv`
Rows: 5,000

| Column                  | Type    | Description                                                                                                            |
| ----------------------- | ------- | ---------------------------------------------------------------------------------------------------------------------- |
| incident_id             | VARCHAR | Surrogate key in format INC-00001. Primary key for this dimension                                                      |
| incident_type           | VARCHAR | Type of network incident. One of: power_outage, congestion, equipment_failure, weather_damage, fiber_cut, interference |
| severity                | VARCHAR | Incident severity level. One of: low, medium, high, critical. ~40% low, ~35% medium                                    |
| resolution_time_minutes | INTEGER | Time to resolve the incident in minutes. Ranges from 15 (low severity) to 2880 (critical, 48 hours)                    |
| affected_towers_count   | INTEGER | Number of towers affected. Ranges from 1 (equipment_failure) to 50 (fiber_cut)                                         |

## FACT_NETWORK_EVENTS — Generated Fact Table

Source: Generated by `src/generator/generate_fact_network_events.py`
File: `data/generated/fact_network_events.csv`
Rows: ~1,000,000
Input: all five dimension tables above

| Column          | Type    | Description                                                                                                                    |
| --------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------ |
| event_id        | VARCHAR | Surrogate key in format EVT-0000001. Primary key for this table                                                                |
| tower_id        | VARCHAR | Foreign key to DIM_TOWERS                                                                                                      |
| customer_id     | VARCHAR | Foreign key to DIM_CUSTOMERS                                                                                                   |
| time_id         | VARCHAR | Foreign key to DIM_TIME                                                                                                        |
| weather_id      | VARCHAR | Foreign key to DIM_WEATHER                                                                                                     |
| incident_id     | VARCHAR | Foreign key to DIM_INCIDENTS. Null for ~95% of events                                                                          |
| rsrp_dbm        | FLOAT   | Reference Signal Received Power in dBm. 4G range: -110 to -80. 5G range: -105 to -75. Degrades under bad weather and incidents |
| sinr_db         | FLOAT   | Signal to Interference plus Noise Ratio in dB. 4G range: 0 to 20. 5G range: 5 to 30                                            |
| throughput_mbps | FLOAT   | Data throughput in Mbps. 4G range: 5 to 150. 5G range: 50 to 900. Drops under congestion and bad weather                       |
| latency_ms      | FLOAT   | Round-trip latency in milliseconds. 4G range: 20 to 60. 5G range: 1 to 10. Increases under incidents                           |
| handover_flag   | BOOLEAN | True when the device handed over to a different tower during this interval. Probability increases in bad weather               |
